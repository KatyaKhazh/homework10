# PoC для CVE-2020-1938 (Ghostcat)

## Описание уязвимости

CVE-2020-1938 (Ghostcat) — уязвимость в Apache Tomcat, связанная с протоколом AJP (Apache JServ Protocol).

По умолчанию Tomcat использует порт 8009 для взаимодействия через AJP.  
Если порт 8009 открыт извне и не настроены ограничения доступа, злоумышленник может:

- получить доступ к файлам веб-приложения,
- выполнить include JSP-файлов,
- потенциально добиться удалённого выполнения кода.

Уязвимы версии Apache Tomcat 6.x, 7.x, 8.x и 9.x до выхода исправлений.


## PoC-скрипт (эмуляция)

Скрипт выполняет проверку доступности порта 8009 (AJP).  

Если порт открыт, выводится сообщение о потенциальной уязвимости.  
Если порт закрыт — уязвимость не подтверждается данной проверкой.

## Пример результата (порт открыт)
Проверяем доступность AJP-порта: 127.0.0.1:8009

[+] Порт 8009 (AJP) открыт.
[!] РЕЗУЛЬТАТ: Потенциально уязвимо (CVE-2020-1938), если используется уязвимая версия Tomcat и AJP доступен без ограничений.
